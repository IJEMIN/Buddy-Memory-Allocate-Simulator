------------ VERSION 0.5 -------------------
*2014/10/31 작성*

1. 개요
"BuddyAllocater"는 메모리를 가정하고, 2^n 단위로 블럭을 나눠, 그 안에 요청된 메모리를 할당한다.
KB단위로 메모리를 할당하는 것을 가정하며, 요청된 메모리를 수용할 수 있는 적절한 2^n 크기의 블록안에 할당한다.
메모리 해제 요청시, 적절한 2^n 크기 단위로만 해제된다.


2. 기능

인트로- 피지컬 메모리의 크기를 입력받는다.

피지컬 메모리를 정할 때 입력단위는 메가 바이트이다.

/*/이후 모든 연산은 KB단위이므로, 시뮬레이션의 용이성을 위해 피지컬 메모리의 단위는 512MB보다 작게 설정하는 것을 추천한다.

출력- 현재 가용 메모리를 표시한다.

콘솔을 통해 숫자를 입력시 해당 기능이 실행된다.
1- 메모리를 할당한다. 할당하고자 하는 크기를 입력받는다.
2- 메모리를 해제한다. 해제하고자 하는 크기를 입력받는다.
0- 프로그램을 종료한다.


3. 세부기능(구현)

AskAllocation
호출시 콘솔을 통해 입력을 받는다.
처음 정의된 전체 피지컬 메모리양과, 가용 가능 메모리 양을 넘어선 입력을 요청받으면 Allocater 호출없이 종료된다.
또한 최소 하한선은 2단위 보다 적은 입력 또한 처리하지 않고 종료한다.


Allocater
AskAllocation와 RealseMemory을 통해서만 호출된다. 요청된 메모리의 크기를 입력 받아 호출되어야 한다.
입력받은 메모리보다 크면서, 그중에서는 가장 작은 2^n값을 검사한다.
처음 하한선과 상한선은 각각 2^1와 2^2로 정의된다.
재귀를 통해 각각 2배수 증가한다.

재귀를 통한 검사중, 요청된 메모리 크기가 하한선 초과, 상한선 이하인 경우
상한값(요청 메모리 보다 큰 가장 작은 2^n값)을 호출함수로 보내고, 재귀를 종료한다.

RealseMemory
메모리 할당 해제 요청을 받아들여, 메모리 할당 해제를 하고자 하는 크기를 입력받는다.
해제를 요구하는 메모리 크기가 현시점에서 할당된 블럭들 보다 큰 경우, 오버플로우 신호를 보내고 종료한다.
할당하고자 하는 메모리가 존재하는 블럭은 그보다 큰 값들 중에서 가장 작은 2^n 크기를 가지므로,
Allocater를 통해 할당하고자 하는 메모리가 존재하는 블럭의 크기를 계산한후 해제한다.



4. 알려진 문제

초기에 피지컬 메모리의 크기를 정하는 입력은 MB단위이다.
이를 KB단위로 변환한다. 따라서 초기 입력에 1024를 곱한 값이 double 변수의 한계값보다 적은 값이 되어야 한다.


5. 추가 안내 사항

메모리를 가정하고 실행하는 시뮬레이션으로서 동작한다.

실제 메모리를 조작하고자 할때는 포인터 변수를 생성한후 KB와 변수의 크기 사이의 단위 변환을 고려하여 배수를 취하여 Allocater가 리턴하는 값만큼
malloc(할당시), free(릴리즈시) 등의 메모리 함수를 통해 할당, 해제를 시행해주는 스크립트를 RelaseMemory, Allocater에 추가하면된다.
